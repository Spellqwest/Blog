# AI Policy

To prevent an overuse of AI during the project, we've set clear boundaries for the use of AI.

## Guidelines

- **Understand your code**: You should be able to explain the code you write and understand the code you write.
- **AI as a coach, not as a coworker**: AI should be used to help you understand the code and learn new concepts, not to write the code for you.
- **AI for optional features**: We may use AI to generate code and other content we view as optional and are not able to implement ourselves. For example, AI could be used to write the code for a shader or a [sound effect](https://strudel.cc), which does not effect the core game and functionality.
